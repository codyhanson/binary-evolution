Restored the simpler accum_test as accum_test_simple.

_simple inits LEDs to 1111, then does an initial accumbytes on addr 0x1510 to get expected value. Stores that val in the LEDs. Writes that pixel to gray. Then loops, re-calculating the accum sum and comparing to the expected. Continues loopin so long as the value is correct. If it falls through, a glitch occurred and that value will be written to the LEDs. Program then halts
